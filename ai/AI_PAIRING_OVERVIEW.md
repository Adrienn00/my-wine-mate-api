# AI Pairing Overview

This document is a short project-facing summary of the AI-based wine and recipe pairing module.

## Goal

The goal of this module is to recommend:

- wines for a selected recipe
- recipes for a selected wine

The recommendation is generated by a trained machine learning model, not by a fixed scoring formula at runtime.

## Core Idea

The system uses an `XGBoost` classifier trained on structured wine-recipe pairing data.

The training data is built from two sources:

- expert pairing rules stored in the `pairingrules` collection
- user feedback stored in the `pairingfeedbacks` collection

This makes the system hybrid:

- it starts from curated domain knowledge
- then it can improve over time from real user feedback

## Main Components

### 1. Feature extraction

Implemented in:

- [pairing_common.py](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/pairing_common.py)

This file extracts structured signals from wines and recipes, such as:

- wine type
- wine style
- alcohol bucket
- sweetness
- body
- acidity
- tannin
- grape varieties
- flavour notes
- recipe category
- main ingredients
- meat type
- spice level
- spices
- cooking method
- texture
- sauce type

These signals are converted into ML feature rows.

### 2. Knowledge base

Implemented in:

- [pairing_knowledge_base.json](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/pairing_knowledge_base.json)
- [seed_pairing_knowledge.py](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/seed_pairing_knowledge.py)

The knowledge base stores expert pairing rules such as:

- good pairings
- bad pairings
- confidence level
- score
- wine-side criteria
- food-side criteria

These rules are inserted into the MongoDB `pairingrules` collection.

### 3. Dataset generation

Implemented in:

- [generate_pairing_dataset.py](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/generate_pairing_dataset.py)

This script:

- reads confirmed wines and recipes from MongoDB
- matches them against the pairing knowledge base
- generates labeled wine-recipe pairs
- optionally merges saved user feedback

If feedback exists for a given pair, it overrides the rule-based label for that pair.

Generated artifacts:

- `pairing_kb_dataset.csv`
- `pairing_kb_dataset.json`
- `pairing_kb_dataset_summary.json`

### 4. Model training

Implemented in:

- [train_pairing_model.py](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/train_pairing_model.py)

This script:

- builds a training dataset
- vectorizes feature dictionaries
- trains an `XGBoost` model
- evaluates the model
- saves the trained model and metrics

Generated artifacts:

- `xgboost_pairing_model.joblib`
- `training_metrics.json`
- `test_predictions_preview.json`

### 5. Inference

Implemented in:

- [recommend_pairings.py](/home/adrienn/allamvizsga/backend/my-wine-mate-api/ai/recommend_pairings.py)

This script loads the trained model and returns:

- top wines for a recipe
- top recipes for a wine

## User Feedback Loop

Implemented in:

- [pairingFeedback.model.js](/home/adrienn/allamvizsga/backend/my-wine-mate-api/src/pairing/pairingFeedback.model.js)

Users can mark a recommendation as:

- `Good match`
- `Bad match`

This feedback is stored in MongoDB and can later be used as real supervised training data.

## Why This Counts as AI

This module is AI-based because:

- it contains a machine learning training step
- it uses a trained model artifact
- runtime recommendation is produced by model inference
- the model can be retrained with new data

So the recommendation is not a hardcoded static rule engine anymore, even if expert rules are still used to bootstrap the first dataset.

## Current Limitations

- recommendation quality still depends strongly on the quality of the expert pairing rules
- user feedback volume is still low
- the model still needs further fine-tuning and retraining
- there is no large manually labeled ground-truth dataset yet

## Recommended Next Steps

- collect more user feedback
- expand the pairing knowledge base
- retrain the model regularly
- export feedback-based datasets for analysis
- add stronger post-filters for obviously bad pairings
